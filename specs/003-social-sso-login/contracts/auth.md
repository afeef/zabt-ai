## New Endpoints Required from Backend

### 1. Initiate Social OAuth Flow

```
GET /auth/oauth/{provider}/authorize
```

**Path parameters:**

| Parameter | Type | Values |
|-----------|------|--------|
| provider | string | `google` \| `microsoft` \| `apple` |

**Query parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| redirect_uri | string | yes | `{FRONTEND_URL}/auth/callback/{provider}` |
| state | string | yes | CSRF token generated by frontend, validated by backend |

**Response:** `302 Redirect` → provider's authorization URL

**Frontend usage** — browser navigation (not an Axios call):
```
window.location.href = `${API_URL}/auth/oauth/${provider}/authorize?redirect_uri=...&state=...`
```

---

### 2. OAuth Callback (backend-handled)

```
GET /auth/oauth/{provider}/callback
```

Called by the provider after authentication. Backend exchanges code for token, then:

**Success — new user (auto-register):**
Silently creates Meetily account from provider email, issues JWT.
→ Redirects to `{FRONTEND_URL}/auth/callback/{provider}?token={jwt}`

**Success — existing social user:**
Retrieves existing account, issues JWT.
→ Redirects to `{FRONTEND_URL}/auth/callback/{provider}?token={jwt}`

**Conflict — email belongs to email/password account:**
→ Redirects to `{FRONTEND_URL}/auth/callback/{provider}?error=email_conflict`

**Failure — cancelled, denied, or provider error:**
→ Redirects to `{FRONTEND_URL}/auth/callback/{provider}?error={error_code}`

**Error codes:**

| Code | Meaning |
|------|---------|
| `email_conflict` | Email already registered via email/password; no linking performed |
| `access_denied` | User cancelled or denied provider permissions |
| `provider_error` | Provider returned an unexpected error |
| `invalid_state` | CSRF state mismatch; possible replay attack |

### 3. SSO Domain Lookup

```
POST /auth/sso/lookup
```

**Request body:**
```json
{ "email": "user@company.com" }
```

**Response 200 — SSO not configured:**
```json
{
  "sso_enabled": false,
  "redirect_url": null,
  "organisation_name": null
}
```

**Response 200 — SSO configured:**
```json
{
  "sso_enabled": true,
  "redirect_url": "https://sso.company.com/saml/...",
  "organisation_name": "Acme Corp"
}
```

**Response 422 — Invalid email:**
```json
{ "detail": "Invalid email address" }
```

**Frontend behaviour on each outcome:**

| Outcome | Frontend action |
|---------|----------------|
| `sso_enabled: true` | `window.location.href = redirect_url` |
| `sso_enabled: false` | Proceed with email/password submit |
| Network error / timeout / 5xx | Proceed with email/password submit (fallthrough) |
| 422 | Show inline validation error on email field |

### 4. Email/Password Login with Remember Me (existing — extended)

```
POST /login/access-token
Content-Type: application/x-www-form-urlencoded
```

**Body (extended):**
```
username=user@example.com&password=secret&remember_me=true
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| username | string | yes | User's email address |
| password | string | yes | User's password |
| remember_me | boolean | no | Default false; controls token lifetime |

**Response 200:**
```json
{ "access_token": "eyJ...", "token_type": "bearer" }
```

**Response 401:**
```json
{ "detail": "Incorrect username or password" }
```

**Backend behaviour** (out of scope — documented for contract clarity):
- `remember_me=false` or absent → token expires in 1 hour
- `remember_me=true` → token expires in 30 days

## Frontend-Only Routes

### OAuth Callback Page

```
GET /auth/callback/{provider}?token={jwt}
GET /auth/callback/{provider}?error={code}
```

Next.js route at `app/auth/callback/[provider]/page.tsx`.

**On `?token=`:**
1. `setToken(token)` — stores JWT in localStorage
2. `router.replace("/")` — clears token from URL, navigates home

**On `?error=email_conflict`:**
Display: "An account with this email already exists. Sign in with your password instead."
Include a link back to `/login`.

**On `?error=` (other):**
Display a generic error message.
Include a link back to `/login`.

**On neither:**
`router.replace("/login")`

### Forgot Password Placeholder

```
GET /forgot-password
```

Next.js route at `app/forgot-password/page.tsx`.
Renders a static placeholder: "Password reset coming soon."
No backend call. No form.
