# Implementation Plan: Export Summary as PDF

**Branch**: `001-export-summary-pdf` | **Date**: 2026-03-04 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-export-summary-pdf/spec.md`

## Summary

Adds a "Download as PDF" option to the existing summary options menu on the meeting detail page. PDF generation is entirely client-side using `pdfmake`, which converts the markdown summary into a text-searchable, formatted PDF document including meeting metadata (title, date, duration, speakers). No backend changes are required.

## Technical Context

**Language/Version**: TypeScript 5 / Node.js 20
**Primary Dependencies**: Next.js 16, React 19, `pdfmake` (new), `@types/pdfmake` (new dev dep)
**Storage**: N/A — no data model or storage changes
**Testing**: Playwright/Python (E2E), TypeScript `tsc --noEmit` (type check)
**Target Platform**: Browser (client-side only; `'use client'` directive)
**Project Type**: Web application — frontend only
**Performance Goals**: PDF download initiates in under 3 seconds for typical meeting summaries
**Constraints**: Must not require server round-trips; pdfmake must be loaded only on client
**Scale/Scope**: Single feature — one new utility module + one menu item addition
