# Implementation Plan: Migrate Auth to Supabase Cloud & Repurpose Backend

**Branch**: `001-migrate-supabase` | **Date**: 2026-02-21 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-migrate-supabase/spec.md`

## Summary

Migrate the system from using a local Logto Docker deployment to the Supabase Cloud Service for authentication. Consequently, decouple the FastAPI backend from auth handling, re-architecting it as purely an AI-processing resource server that secures its routes by verifying Supabase JWTs.

## Technical Context

**Language/Version**: Python 3.11 for backend, TypeScript (React/Next.js) for frontend
**Primary Dependencies**: FastAPI, python-jose, @supabase/supabase-js, @supabase/ssr
**Storage**: Supabase PostgreSQL
**Testing**: Playwright/Python (E2E)
**Target Platform**: Web Frontend + Linux Backend
**Project Type**: Web application (frontend + backend)
**Performance Goals**: Reduce local CPU/RAM load by <200MB, decrease startup time
**Constraints**: Must remove all references to Logto in both systems and Docker.
**Scale/Scope**: System-wide Auth substitution
