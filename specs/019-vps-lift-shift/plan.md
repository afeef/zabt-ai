# Implementation Plan: Lift-and-Shift Backend to Contabo VPS

**Branch**: `019-vps-lift-shift` | **Date**: 2026-03-07 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/019-vps-lift-shift/spec.md`

## Summary

Migrate the entire Zabt backend Docker Compose stack from the developer's local machine to a Contabo VPS (6 vCPU, 12GB RAM). The worker runs on CPU on the VPS (slower but functional). Add Qdrant to docker-compose for future use. Configure Cloudflare tunnel on the VPS. Keep the same `docker-compose.yml` working for both local dev and VPS production via environment variable toggles. No application code changes.

## Technical Context

**Language/Version**: Python 3.11 (backend), TypeScript / Node.js 20 (frontend-2), YAML (Docker Compose, Kong, Cloudflare)
**Primary Dependencies**: Docker Compose, Cloudflare Tunnel (`cloudflared`), Kong 3.6, PostgreSQL 16, Redis 7, MinIO, Qdrant, Celery
**Storage**: PostgreSQL (via SQLModel), MinIO (S3-compatible), Redis (Celery broker), Qdrant (vector DB, empty for now)
**Testing**: Manual E2E validation (upload → transcribe → summarize → email)
**Target Platform**: Linux x86_64 (Contabo VPS, Ubuntu 22.04/24.04)
**Project Type**: Infrastructure migration (no application code changes)
**Performance Goals**: Page loads <3s, file uploads up to 500 MB, CPU transcription 20-60 min per 30 min audio (acceptable for this phase)
**Constraints**: 6 vCPU, 12 GB RAM, <8 GB RAM usage under normal load
**Scale/Scope**: Single VPS, single-user to low-traffic usage
